## Project Overview

**ZPA Status Mini-SIEM** — A lightweight syslog collector, reporting tool, and web dashboard for Zscaler Private Access (ZPA). Receives ZPA session logs via syslog, generates daily Excel and JSON reports on user activity, and provides a web dashboard for historical access and search.

Target platform: **Red Hat Enterprise Linux 9/10**.

## Architecture

```
[ZPA App Connector] --syslog(TCP/514)--> [rsyslog] --> /var/log/zpa/zpa.log
                                                              |
                                         (daily timer) --> [report_generator.py]
                                                              |
                                              ┌───────────────┼───────────────┐
                                              v               v               v
                                        Excel report    JSON report     Share upload
                                        (download)      (dashboard)     (SMB/SCP)

[Web Dashboard] <--HTTPS--> [Flask app] --> reads JSON reports
```

- **Syslog Collection**: rsyslog (RHEL built-in) receives ZPA logs on TCP/514 and writes to `/var/log/zpa/zpa.log`.
- **Log Rotation**: logrotate handles daily rotation with 30-day retention, gzip compression, dateext naming.
- **Session Parser** (`src/session_parser.py`): Shared module for log parsing, session consolidation, and merge logic.
- **Report Generator** (`src/report_generator.py`): Runs daily (via systemd timer), generates Excel + JSON reports, uploads to file share, cleans up old reports.
- **Web Dashboard** (`src/web_dashboard.py`): Flask HTTPS app with basic auth, report browsing, user search, and Excel download.
- **Share Upload** (`src/share_upload.py`): Uploads reports to SMB/CIFS or SCP shares.
- **Config** (`src/config.py`): Reads `/opt/zpa-siem/config.ini` (generated by installer).

## Tech Stack

- **Python 3.12** (RHEL 10.1 system Python)
- **Flask** for web dashboard (HTTPS, session auth)
- **openpyxl** for Excel generation
- **bcrypt** for password hashing
- **python-dotenv** for dev configuration
- **rsyslog** (RHEL built-in) for syslog reception
- **logrotate** (RHEL built-in) for log rotation

## Commands

```bash
# Install on RHEL (as root, interactive)
sudo bash install.sh

# Manage installation
sudo bash install.sh --configure   # Reconfigure
sudo bash install.sh --status      # Show status
sudo bash install.sh --uninstall   # Remove

# Generate report manually
sudo /opt/zpa-siem/venv/bin/python3 /opt/zpa-siem/report_generator.py

# Development (local)
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
PYTHONPATH=src python3 src/report_generator.py --log-file logs/zpa-sample.log --output-dir reports/
PYTHONPATH=src python3 src/web_dashboard.py

# Build distribution zip
bash build-dist.sh

# Run tests
python3 -m pytest tests/
```

## Deployment Model

**Critical constraint**: the team has NO access to the production server. All deployment is done via `install.sh` executed by another team. The install script must be:
- **Interactive**: clear prompts with sensible defaults
- **Idempotent**: safe to run multiple times
- **Self-documenting**: generates documented config.ini
- **Reconfigurable**: `--configure` mode for post-install changes

### What `install.sh` configures:
1. rsyslog: drop-in config to receive ZPA syslog on TCP/514, write to `/var/log/zpa/`
2. logrotate: daily rotation, 30-day retention, gzip compression
3. Firewall: open TCP/514
4. Python venv + dependencies in `/opt/zpa-siem/` (vendored or online)
5. HTTPS self-signed certificate for dashboard
6. systemd timer for daily report generation
7. systemd service for web dashboard (optional)
8. Configuration file with all settings documented

## Session Processing Logic

1. Filter: only `zpn_client_type_zapp` (real user sessions)
2. Discard: auth probes (duration < 5 seconds)
3. Merge: consecutive sessions with gap <= 60 seconds
4. Convert: timestamps from UTC to configured timezone
5. Output: one row per consolidated session

## ZPA Log Format

ZPA logs are JSON payloads in syslog lines. Key fields used:
- `Username`, `SessionID`, `SessionStatus`
- `TimestampAuthentication`, `TimestampUnAuthentication`
- `ClientType`, `PublicIP`, `PrivateIP`
- `City`, `CountryCode`, `Hostname`, `Platform`
- `Version`, `TrustedNetworksNames`
- `TotalBytesRx`, `TotalBytesTx`
